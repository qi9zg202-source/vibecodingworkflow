Session 0 — 设计阶段
目标：产出所有稳定规划文档，不写任何业务代码
Session 0 分两步：0a（需求阶段）产出 CLAUDE.md / task.md / PRD.md，用户确认后；0b（规划阶段）在同一窗口继续，产出 design.md / work-plan.md / tasksubsession1~N.md / memory.md。
v2.4 新增 — Session 0a 三个子步骤（全部在同一窗口内完成）：

· Step 1a — 问卷收集：填写项目背景、功能范围、对标目标、客户资料路径
· Step 1b — 背景搜索：Agent 自动搜索客户基本信息、对标行业地位、对标最佳实践，输出摘要等待确认 → 写入 CLAUDE.md Benchmark Reference section
· Step 1c — 资料读取：分段读取客户资料文件（Excel/PDF），提取点位、单位、采集频率等 → 写入 CLAUDE.md Domain Data section

三步确认后，PRD.md 新增 Design Standard section，明确设计必须达到或超越对标水准。

什么情况算"大改"？

满足任一条件即视为大改：
· In Scope 有模块增减（新增或删除功能模块）
· 已完成 Session 的交付物与新需求不兼容
· 核心数据模型或交互流程根本性变化

常见问题
遇到问题先看这里
我需要提前准备什么？
不需要。只需知道"做什么系统"和"这次实现哪个功能"，Agent 会通过问答帮你整理。
测试通过了，Agent 会不会自动写 summary？
不会。测试通过后 Agent 只输出结果并等待你验收。只有你明确回复"通过"/"继续"后，Agent 才会写 artifacts/session-N-summary.md 并更新 memory.md。测试通过 ≠ Session 完成，你的验收确认 = Session 完成。
每个 Session 必须开新对话窗口吗？
是的。每个 Session 对应独立的对话窗口。新窗口保证 Agent 以干净状态读取 tasksubsession 文件，避免旧对话上下文干扰执行。
Session 测试失败了怎么办？
Agent 会告知失败原因，memory.md 不更新。你可以修改 tasksubsessionN.md 后重新执行，或直接让 Agent 修复后重跑测试。
大改后，已完成的 Session 会被全部推倒重做吗？
不会。Agent 会输出复用评估表，逐条评估每个已完成 Session 的交付物与新需求的兼容性（✅ 兼容 / ⚠️ 部分兼容 / ❌ 不兼容），你逐条确认后，Agent 只重做你标记为"重做"的 Session，其余复用。
我想修改 CLAUDE.md，Agent 会怎么处理？
Agent 会先提醒你"CLAUDE.md 通常不改，只有项目级约束发生根本变化时才需要更新"，然后明确确认。只有你确认"需要"且确属项目级约束根本变化时，才会进入大改流程。
最终 HTML 的模拟数据从哪里来？
Agent 从 CLAUDE.md 的业务背景中提取行业、角色、场景等信息，结合 PRD.md 的功能范围生成。你在 Session 0 填写的业务背景越详细，模拟数据越贴近真实。
只有一个文件 1paperprdasprompt.md 就够了吗？
是的。这是单文件交付模式的核心设计——客户只需获取这一个文件，大模型即运行时，零依赖。
Session 0b 执行到一半中断了怎么办？
重新发送启动指令，Agent 会检测到 design.md 已存在但 work-plan.md 不存在，自动从 work-plan.md 开始继续产出，不会覆盖已有的 design.md。
CLAUDE.md 里为什么没有工作流规则了？
v2.2 版本将工作流规则移到了 1paperprdasprompt.md 的 SECTION 4，CLAUDE.md 只保留项目业务上下文。这样每个 Session 读取 CLAUDE.md 时不会被工作流规则占用上下文空间。
对标目标有什么用？
对标目标（如台积电）用于两个地方：一是 Agent 在 Step 1b 搜索该对标企业的行业地位和最佳实践，写入 CLAUDE.md 的 Benchmark Reference section，作为设计参考事实；二是 PRD.md 的 Design Standard section 会明确要求"设计必须达到或超越对标水准"，让评审方看到设计依据。填写越具体（如"台积电制冷系统能效管理"），搜索结果越精准。
客户资料文件太大读不完怎么办？
Agent 会采用分段读���策略：结构化文件（Excel/CSV）优先读取表头和前 50 行，提取字段名、单位、采集频率等元数据；非结构化文件（Word/PDF）按章节分段读取，每段确认后继续。如果文件超出单次读取限制，Agent 会告知已读取的范围和提取结果，你可以决定是否继续读取剩余部分。核心目标是提取"有哪些数据、数据结构是什么"，而非读完全部原始数据。
Roo Code /run-session 和手动发送指令有什么区别？
效果相同，触发方式不同。/run-session 会自动读取 memory.md 解析 next_session，省去手动确认当前进度的步骤。手动方式需要你自己判断该执行哪个 tasksubsession 文件。两种方式都支持，可以混用。正常推进时用 /run-session 更省心；需要跳过某个 Session、重新执行特定 Session 或执行 Session 0b 重新规划时，建议手动触发。
