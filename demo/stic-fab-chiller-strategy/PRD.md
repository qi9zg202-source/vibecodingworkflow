# PRD.md

## Problem
- 机械课当前对高效制冷机房的运行分析、策略建议和执行反馈分散在报表、Excel、群消息和经验判断中。
- 核心指标虽然能从不同系统获取，但缺少统一的趋势分析和异常识别入口。
- 策略建议难以形成标准化策略包，执行后缺少反馈和稳态验证闭环，导致经验难沉淀。

## Goal
- 提供一个高效制冷机房策略优化建议管理功能，统一承载指标监控、策略包管理和执行闭环。
- 降低机械课工程师识别异常、形成策略、追踪执行结果的时间成本。
- 让策略优化建议可复盘、可评估、可回退。

## Users
- 机械课运行工程师
- 节能优化工程师
- 厂务管理者 / 审核者

## Core User Value
- 看到关键指标趋势和异常预警
- 快速定位适用的策略包
- 追踪策略从建议到落地再到稳态验证的全过程

## Functional Scope
1. 指标曲线监控分析
2. 优化策略包
3. 策略包下发、反馈、稳态验证

## Acceptance Criteria
- 支持查看 EER、COP、负荷、电量、温差、预警等指标趋势
- 支持策略包列表和详情查看
- 支持策略包状态流转：待评审、待下发、执行中、反馈中、稳态验证、已完成、已回退
- 支持记录反馈、稳态观察结论和回退原因

## Non-Goals
- 不做实时自动控制
- 不替代底层厂务控制系统
- 不在本功能中引入全厂统一策略引擎
