# PRD.md

## Problem
- Fab CUS 制冷站的运行分析、策略判断和执行复盘分散在 historian 报表、Excel 台账、群消息和工程师经验中，缺少统一入口。
- 机械课在做节能优化时，经常需要同时考虑中温环路、低温环路、热回收、湿球条件、电价时段和 N+1 冗余，但现有工具很难把这些约束放在同一工作流里。
- 策略建议经常停留在会议结论，缺少“适用条件 -> 风险 -> 审批 -> 执行 -> 稳态验证 -> 回退”的标准闭环，导致经验无法形成复用资产。

## Goal
- 提供一个贴近真实厂务业务的策略工作台，统一承载制冷站监控、策略包管理和执行闭环。
- 让工程师在形成策略建议时，同时看到负荷基线、约束边界、预期收益和风险等级。
- 让每次策略执行都可审批、可跟踪、可回退、可沉淀为后续业务样板。

## Users
- 值班运行工程师：关注当前负荷、异常、待执行任务和回退风险
- 节能优化工程师：关注工况匹配、策略包选择、ROI 范围和验证计划
- 课长 / 审批人：关注风险、冗余影响、执行结果和审计记录
- 设备可靠性 / 工艺接口人：关注温边界、最小流量、稳态观察和异常处置

## Core User Value
- 在同一页面理解“当前运行状态 + 可以考虑的策略 + 风险与回退条件”
- 快速从历史工况中选择适用策略包，而不是临时拼接经验
- 清楚知道某次策略是否审批通过、是否执行、是否达成稳态、为什么回退

## Business Scenarios
1. 冬季低湿球场景：优先评估自然冷却 / 换热协同策略
2. 过渡季中等负荷场景：评估混合模式与机组排序优化
3. 夏季高负荷场景：在 N+1 约束下做安全优先的节能建议
4. 异常工况场景：传感器缺数、单台机组异常、热回收链路不可用时的降级策略

## Functional Scope
1. 制冷站运行概览
2. 负荷基线与工况分段
3. 策略包目录与详情
4. ROI / 风险 / 回退条件展示
5. 审批、下发、反馈与稳态验证
6. 异常工况与审计记录

## Acceptance Criteria
- 支持查看供回水温、流量、负荷、机组台数、EER/COP、电量、湿球和告警摘要
- 支持按季节、负荷和工况查看策略包及其适用边界、预期收益、风险等级和回退条件
- 支持从策略建议进入审批与执行闭环，并完整记录执行反馈和稳态结论
- 支持审计日志与角色权限，保证审批和回退动作不可绕过
- 支持冬季 / 过渡季 / 夏季 3 类业务场景验证和高风险边界检查

## Non-Goals
- 不直接替代厂务控制系统
- 不做实时自适应控制算法上线
- 不建设跨厂区统一调度平台
- 不在本 demo 中接入真实数据库或身份系统
